# CLAUDE.md

## FSD 아키텍처 가이드

이 저장소의 FSD 기준 문서는 `FSDguide.md`를 단일 소스로 사용합니다.

### 핵심 원칙

- 의존성은 상위 레이어에서 하위 레이어로만 향합니다.
- `pages`는 조립 역할만 수행하고 비즈니스 로직은 `features`/`entities`에 둡니다.
- 재사용 가능한 공용 코드는 `shared`로 이동합니다.

### 레이어 책임

- `app`: 엔트리, 라우팅, 전역 레이아웃/스타일
- `pages`: 라우트 단위 화면 조립
- `widgets`: 여러 페이지에서 재사용되는 큰 UI 블록
- `features`: 사용자 행동 중심 시나리오
- `entities`: 도메인 모델/타입/UI/API 어댑터
- `shared`: 공용 UI, 훅, 유틸, 타입, 기본 클라이언트

자세한 규칙과 현재 구조는 `FSDguide.md`를 확인합니다.
